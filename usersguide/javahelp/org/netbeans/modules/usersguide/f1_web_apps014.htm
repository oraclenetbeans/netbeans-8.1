web.xml Visual Editor: Login Configuration

Use the Login Configuration section of the web.xml Visual Editor to configure authentication for your web application.

Element Description

None

 No authentication, this is the equivalent of removing the login configuration element from the deployment descriptor.

Digest

 An authentication mechanism in which a Web application authenticates itself to a Web server by sending the server a message digest along with its HTTP request message. The digest is computed by employing a one-way hash algorithm to a concatenation of the HTTP request message and the password of the client. The digest is typically much smaller than the HTTP request and does not contain the password.

Client Certificate

 An authentication mechanism that uses HTTP over SSL, in which the server and, optionally, the client authenticate each other with a public key certificate that conforms to a standard that is defined by X.509 Public Key Infrastructure.

Basic

 You can type the name of the security realm in the Realm Name field. There is no validation for this value.

Form

 Lets you select JSP pages for the login and error pages.
