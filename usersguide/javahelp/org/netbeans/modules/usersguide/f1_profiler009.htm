New Profiling Point Wizard: Take Snapshot

Use to take profiling snapshots or heap dumps each time that the profiling point is hit.

Element Description

Name

 Specifies the name that you want to use to identify the profiling point.

This name is used to identify the profiling point in the Profiling Points window.

Settings

 Snapshot settings

    Take

 Specifies the type of snapshot that you want to take.

    Save

 Specifies the location for the saved snapshots.

You can save snapshots to the default directory in your project or you can specify a location on your local system.

    Reset results after taking snapshot

 Resets the collected profiling results after each snapshot is taken.

Location

 Specifies the name of the file and the line in the source code where you want to place the profiling point.

Click Browse to locate the file in your project.

Select Begin or End to specify if the profiling point is hit when a thread hits the beginning of the line or reaches the end of the line.

If you opened the New Profiling Point wizard by clicking in a line in a source file, the file and line are automatically set for you but you can modify the settings as needed.
