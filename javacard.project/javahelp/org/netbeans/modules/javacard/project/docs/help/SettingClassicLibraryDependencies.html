Setting Classic Library Dependencies

The Dependencies panel displays the classic libraries, their paths, and deployment strategies in the project as well as enables you to Add a Classic Library, Remove a Classic Library, change the order of libraries, and Change a Classic Library Deployment Strategy used in the project.

Libraries - If you want to use the same set of sources in multiple projects, you can set it in the project as a classic library and share it among different users and projects as a separate project. There are two types of classic libraries that can be added to a project:

  • Libraries that depend on existing classic library projects.

  • Libraries that depend on existing classic applet JAR files.

    Use a classic library that depends on a JAR file when the JAR file satisfies one of the following conditions:

    • The JAR file represents an API that resides on the target platform but is not deployed.

    • The JAR file contains classes that must be included in the JAR file generated for the library project.
